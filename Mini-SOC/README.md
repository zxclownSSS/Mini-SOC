# Инструкция по развёртыванию и использованию Mini-SOC стенда на базе Wazuh

## 1. Назначение

Mini-SOC стенд предназначен для демонстрации работы системы мониторинга событий информационной безопасности на базе Wazuh.

В рамках проекта реализованы:

* сбор журналов событий Windows;
* централизованный мониторинг событий безопасности;
* визуализация событий через Wazuh Dashboard;
* пользовательские правила обнаружения;
* демонстрационные сценарии работы системы.

---

## 2. Состав стенда

Стенд состоит из двух виртуальных машин:

### WazuhServer

Содержит:

* Wazuh Manager;
* Wazuh Indexer;
* Wazuh Dashboard.

### Windows10-SOC

Содержит:

* Microsoft Windows 10;
* Wazuh Agent.

---

## 3. Системные требования

Для запуска стенда необходимо:

* Oracle VirtualBox версии 7.0 или выше;
* не менее 8 ГБ оперативной памяти;
* не менее 50 ГБ свободного места на диске.

## 5. Развёртывание стенда

### 5.1 Подготовка

1. Установить Oracle VirtualBox версии 7.0 или выше.
2. Распаковать архив проекта в удобное место на диске.
3. Убедиться, что структура проекта сохранена без изменений.

---

### 5.2 Подключение виртуальных машин

#### Добавление WazuhServer

1. Открыть Oracle VirtualBox.
2. Выбрать:

```text
Машина → Добавить
```

3. Указать файл:

```text
VM\WazuhServer\WazuhServer.vbox
```

4. Подтвердить добавление виртуальной машины.

---

#### Добавление Windows10-SOC

1. В Oracle VirtualBox выбрать:

```text
Машина → Добавить
```

2. Указать файл:

```text
VM\Windows10-SOC\Windows10-SOC.vbox
```

3. Подтвердить добавление виртуальной машины.

После выполнения указанных действий обе виртуальные машины должны появиться в списке VirtualBox.

## 8. Проверка работоспособности

После входа в Dashboard открыть раздел:

```text
Endpoints
```

Убедиться, что агент:

```text
Windows-Client
```

находится в состоянии:

```text
Active
```

После этого открыть раздел:

```text
Threat Hunting
```

и убедиться в поступлении событий от агента Windows.

## 9. Демонстрационные сценарии

### Сценарий №1. Создание локальной учётной записи

На виртуальной машине Windows10-SOC выполнить команду:

```cmd
net user eviluser Test123! /add
```

Ожидаемый результат:

В разделе Threat Hunting появляется алерт:

```text
Rule ID: 100002
Custom Alert: New local user account created
```

---

### Сценарий №2. Добавление пользователя в локальную группу

На виртуальной машине Windows10-SOC выполнить команду:

```cmd
net localgroup Users eviluser /add
```

Ожидаемый результат:

В разделе Threat Hunting появляется алерт:

```text
Rule ID: 100003
Custom Alert: User added to Administrarotrs group
```

---

## 10. Пользовательские правила

Пользовательские правила расположены на сервере Wazuh по пути:

```text
/var/ossec/etc/rules/local_rules.xml
```

В рамках проекта реализованы следующие правила:

### Rule ID 100002

Обнаружение создания локальной учётной записи Windows.

---

### Rule ID 100003

Обнаружение добавления пользователя в локальную группу Windows.

---

## 11. Добавление новых правил

Для редактирования файла правил выполнить:

```bash
sudo nano /var/ossec/etc/rules/local_rules.xml
```

После внесения изменений выполнить проверку конфигурации:

```bash
sudo /var/ossec/bin/wazuh-analysisd -t
```

При отсутствии ошибок выполнить перезапуск менеджера:

```bash
sudo systemctl restart wazuh-manager
```

После перезапуска новые правила начнут применяться к поступающим событиям.

---

4. Закрыть Oracle VirtualBox.
